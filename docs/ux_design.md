## 1. 사용자 여정 지도 (Persona-driven Journey Maps)

> 1 단계 페르소나 3 (A·B·C) 와 UC 6 직접 매핑. 페르소나별 5 단계 여정
> (진입 → 종목 선택 → 분석 결과 → 해석 → 한계 인지) — 단계별 *행동·감정·
> 생각* 명시.

### 1.1 페르소나 A — 산출물 평가 담당자 (기술 외 직무)

**매핑 UC**: UC-1 (첫 진입 5 초), UC-6 (메뉴 둘러보기), UC-5 부수 (한계 배지).

| 단계 | 행동 | 감정 | 생각 |
|---|---|---|---|
| 진입 | URL 진입 → 첫 화면 카피 + 예시 종목 카드 확인 (5 초) | 호기심·평가 모드 | "어떤 시스템인가" |
| 종목 선택 | 예시 종목 카드 1 개 클릭 또는 사이드 메뉴 둘러보기 | 가벼움 (직접 검색 부담 0) | "예시 보면 충분" |
| 분석 결과 | 통합 화면 4 카드 + 1 해석 박스 30 초 내 직관 파악 | 안심·인상 형성 | "일반인도 이해 가능하구나" |
| 해석 | 사이드 메뉴 4 페이지 약 1 분/페이지 순회 | 평가 진행 | "전체 가치 파악" |
| 한계 인지 | 페이지 헤더/푸터 한계 배지 발견 (1 회 모달 + 항상 표시) | 신뢰 ↑ | "정직한 시스템" |

**여정 종료 결과**: 5 분 내 *무엇을 만들었고 어떤 가치가 있는가* + 한계
정직 인지. 산출물 평가 *호의적*.

### 1.2 페르소나 B — 금융 입문 일반 투자자

**매핑 UC**: UC-2 (통합 화면), UC-3 (시장 위기 시점), UC-5 (한계 정독).

| 단계 | 행동 | 감정 | 생각 |
|---|---|---|---|
| 진입 | URL 진입 → 검색창 노출 확인 | 목적 명확 | "내 관심 기업 찾기" |
| 종목 선택 | 검색창에 "삼성전자" 입력 → 자동완성 5 개 → 종목 선택 | 빠른 진입 | "있구나" |
| 분석 결과 | 통합 화면 30 초 내 종합 평가 — 4 카드 색상·아이콘 직관 파악 | 의미 파악 즉시 | "이 회사 지금 어떤 상태인가" 답 받음 |
| 해석 | 해석 박스 (LLM 서술) + 시장 상태 페이지 진입 → 위기 시점 hover | 깊은 이해 | "왜 이런 상태인가" + "시장은 어떤 상황인가" |
| 한계 인지 | 첫 진입 모달 → 한계 페이지 정독 약 5 분 | 신뢰·책임감 | "실제 투자에 사용 금지 명시 — 안심" |

**여정 종료 결과**: 1 분 내 *현 시점 종합 평가* + 5 분 내 *해석 + 시장 +
한계 모두 이해*. 본 시스템 *데모로서 신뢰*.

### 1.3 페르소나 C — 비전공 동료·친구

**매핑 UC**: UC-4 (임의 종목 탐색), UC-5 부수 (한계 호기심), UC-6 (메뉴 둘러보기).

| 단계 | 행동 | 감정 | 생각 |
|---|---|---|---|
| 진입 | URL 진입 → 첫 화면 둘러보기 | 호기심 | "친구가 만든 시스템" |
| 종목 선택 | 검색 자동완성에서 *랜덤* 종목 클릭 | 놀이 모드 | "어떤 종목이든 동작하나" |
| 분석 결과 | 통합 화면 로드 시간 측정 (< 3 초 기대) → 다른 종목 추가 시도 | 가벼움 | "빠르고 매끄럽네" |
| 해석 | 해석 박스 + 시장 상태 페이지 짧게 둘러보기 | 짧은 흥미 | "신기한 데이터" |
| 한계 인지 | 한계 페이지 짧게 확인 | 인정 | "정직한 데모" |

**여정 종료 결과**: 5 분 내 *데모로서 매끄러움* + *들 만한 이야기* 확인.

```
대시보드 (page_title = "한국 KOSPI200 기업 분석 데모")

│   ├── 예시 종목 카드 (3 종목, FR-5 매핑)
│   ├── 메뉴 둘러보기 가이드

│   ├── 분석 시점 선택 사이드바

│   ├── 4 핵심 카드 (종목·재무 건강·시장 상태·통합 리스크)

│   └── 다음 단계 link (한계 페이지·다른 종목)
│
├── 시장 상태 (Market State)
│   ├── 시점별 색상 시각화 (위기 강조)
│   ├── 위기 시점 hover/클릭 한국어 설명
│   └── 일반인 해석 박스
│
└── 한계 (Limitations)
    ├── 모델 한계 3 항목 (일반인 표현)
    ├── 데모 명시 + 실제 투자에 사용 금지
    └── 모델 카드 link (reports/d2_baseline_model_card.md ·
        reports/regime_model_card.md — 기술 차원 별도 노출)
```

### 2.3 페이지 간 이동 (UC-6 매핑)

- 개요 → 종목 분석 (CTA + 사이드 메뉴)

- 종목 분석 → 한계 (배지 클릭)
- 시장 상태 → 한계 (사이드 메뉴)
- 한계 → 종목 분석 (사이드 메뉴 또는 "다른 종목 보기")
- 모든 페이지 → 한계 배지 (항상 표시, NFR-4)

## 3. 인터랙션 흐름 (Interaction Flows)

> 종목 선택·분석 시점·호버·클릭·빈 상태·부분 데이터·에러·로딩 8 차원.
> 기존 `§9`·`§10` 의 그대로 재사용 + 부분 변환.

### 3.1 종목 검색·선택 (FR-4 매핑)

- **검색**: 자동완성 (2 자 이상 입력 후 300 ms 내 표시, 5 개 이내)
- **format**: "005930 삼성전자" (코드 + 종목명)
- **기본값**: 예시 종목 3 개 중 1 (FR-5 매핑) 또는 마지막 선택 종목
- **변경 시**: 즉시 페이지 갱신
- **인기 종목 추천**: 사이드바 또는 개요 페이지 (FR-4 보조)

### 3.2 분석 시점 선택 (부분 변환)

- 최신부터 정렬 (recent first)
- 분석 평가 제외 시점은 *시각적 표시* (예: "2019-12-30 ⚠️ 분석 평가 제외")
  → 이유 hover 시 한국어 설명 ("이 시점은 평가 자료 부족으로 분석 제외")
- 변경 시 즉시 갱신

(변환: 기존 "skipped fold — 양성 0 으로 평가 제외" → 일반 차원 표현)

### 3.3 호버 (hover) (부분 변환)

- **차트**: 일자·종가·시장 상태 (한국어 표현)
- **메트릭**: 정의 tooltip (한국어 자연문, 1 줄)
- **한계 배지**: 한계 본문 미리보기

(변환: 기존 "z-score" 노출 → "표준 점수" 일반 표현 또는 *수치 자체만* 표시)

### 3.4 클릭 (click) (그대로 재사용)

- **모델 카드 link**: 새 탭으로 모델 카드 페이지 (기술 차원 별도 노출)
- **한계 배지**: 한계 페이지 진입
- **예시 종목 카드**: 종목 분석 페이지 진입
- **사이드 메뉴**: 페이지 이동

### 3.5 빈 상태 (Empty State) (부분 변환)

- 분석 평가 제외 시점 선택 → "이 시점은 평가 자료 부족으로 분석 제외" 안내
- universe 비멤버 종목 → "이 종목은 분석 시점 범위 밖" 안내
- 데이터 산출물 부재 → 일반인용: "준비 중" 안내 / 개발 모드: 상세 명령
  (단계 6 결정)

### 3.6 부분 데이터 (부분 변환)

- 분석 시작 9 개월간 → "분석 시작 9 개월간은 시장 상태 분류 정확도가 낮음"
  경고 + 차트는 계속 표시
- LLM 서술 부재 (build 단계 미완) → "해석 준비 중" 안내 + 4 카드는 정상
  표시

(변환: 기존 "regime warmup 9 개월" → "분석 시작 9 개월간은 시장 상태 분류
정확도가 낮음" 일반 차원 표현)

### 3.7 에러 상태 (그대로 재사용)

- *명확한 한국어* + *복구 방법* 안내
- traceback 노출 금지 (개발 모드 제외)
- 부분 실패 (예: 일부 차트 렌더 실패) → 다른 섹션은 정상 표시

### 3.8 로딩 상태 (그대로 재사용)

- > 500 ms 작업만 `st.spinner("데이터 로드 중...")` 표시
- 첫 진입 시 모든 데이터 `@st.cache_data`

## 5. UX 원칙 5 원칙

> 1 단계 박제 직접 매핑. 본 5 원칙이 본 UX 설계 전체의 *상위 제약*.

### (1) 일반인 5 초 이해 (NFR-2 매핑)

각 페이지·컴포넌트는 *ML/금융 비전공 일반인이 5 초 내에 그 화면이 무엇을
보여주는지 이해 가능*. 단계 6 QA 에서 *일반인 ≥ 3 명* 직접 검토 통과 의무.

**★ 부속 박제 (자문 보강, 2026-05-22)**:

> "ML 전문 용어 원본 노출 0 의 의미는 *번역어도 ML 전문 용어 흔적이면
> 위반*. 예:
> - 'walk-forward' → '시간 분할 학습' = *번역어가 ML 차원 의미 보존* = 위반
> - 'walk-forward' → '분석 평가 제외 시점' = *ML 차원 의미 자체 사라진
>   표현* = 통과
>
> 일반인 5 초 이해 = ML 차원 의미 자체가 사라진 표현. ML 용어의 *직접
> 번역이 아니라* *일반 차원 의미만* 잔존 표현."

**본 부속 박제 사유**: docs/ux_design.md `§8.2 ML 용어 한국어 번역 표` 폐기
시 *번역 노출도 원본 노출* 정신의 명시 박제. 본 박제가 `§3` 인터랙션 흐름
부분 변환 ("skipped fold" → "분석 평가 제외 시점" / "z-score" → "표준
점수" / "warmup" → "분석 시작 9 개월간") 의 변환 본질 정신 직접 박제.

### (2) 본질 위반 신호 6 항목 회피 (1 단계 §1.2 + §8 매핑)

각 페이지·컴포넌트 작성 시 6 항목 체크리스트 매번 자기 적용 (1 단계 `§8`
박제):

1. ML 전문 용어 원본 노출 0 (`§1` 위반 신호 #1 + 부속 박제 매핑)
2. 전이 행렬·log-likelihood·시드 분산 주역 표시 0 (FR-9 Won't 매핑)
3. 데이터 dump 표·raw 보고서 0
4. 제목·축·범례가 ML 용어로만 = 0 (`§1` + 부속 박제 매핑)
5. 모델 카드 → 시각화 변환만 = 0 (페르소나·시나리오 매핑 보강 의무)
6. 일반인 5 초 검증 통과 (NFR-2 매핑)
