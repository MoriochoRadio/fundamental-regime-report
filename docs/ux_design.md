# UX Design — Streamlit 통합 대시보드

**버전**: v1 (2026-05-21, Phase 2 산출물)
**상태**: 사용자 검토 대기
**관련**: PROGRESS §5.7·§5.9 (자문 검증 누락 학습) / CLAUDE.md §2·§3.3 (사용자 시나리오)

---

## 1. 자문 측 한계 명시 (정직 박제)

본 UX 설계 문서는 **자문 측 추정 기반**. 다음 한계 인지:

- **사용자 인터뷰·관찰 부재** — 페르소나·시나리오는 자문 측이 *프로젝트 동기 + CLAUDE.md §2 박제*로 추론
- **사용자 테스트 부재** — Phase 5 QA 에서 *시각 회귀 + 사용자 시나리오 e2e* 로 *기능 검증* 가능하나 *사용성 평가는 사용자 자체*
- **§5.9 패턴 5번째 학습 자기적용** — 본 문서가 *또 vacuous 가 되지 않도록* 사용자 검토 게이트 + 실제 구현 시 *각 컴포넌트가 페르소나·시나리오에 매핑*되는지 검증

→ 본 문서는 *합리적 baseline*. 사용자 수정·보강 후 Phase 3 진입.

## 2. 프로젝트 핵심 가치 (디자인 원칙 입력)

CLAUDE.md §2 박제 + PROGRESS 누적 박제 종합:

| 가치 | 의미 | UX 함의 |
|---|---|---|
| **정직성 사슬 5 차원** | 변수·양성충분성·격리·시간·LLM 격리 (CI 강제) | "한계" 페이지가 부속이 아닌 *핵심 가치* — 시각적 강조 |
| **Negative finding 정량 박제** | PR-AUC 0.0136 < base rate (KOSPI200 희소성 증거) | 결과 페이지가 *모델 자랑*이 아닌 *데이터 한계 증거* — 그래픽 hierarchy |
| **국면 조건부 해석** (§3.3) | 같은 부채비율도 위험회피 국면에서 더 강한 신호 | 종목 분석 페이지의 *해석 박스*가 본질 — 단순 metric dump X |
| **방법론적 엄밀성** | walk-forward + embargo + 격리 + 룩어헤드 차단 | 페이지 하단 *메서드 박스* — 의심 시 즉시 확인 가능 |
| **K=4 ablation 정량 발견** | log-lik 개선 + 코로나 위기 27.9%→82% | regime 페이지가 *발견 스토리* 전달 |

→ UI 의 모든 결정은 *위 5 가치 중 어느 것에 기여하는가* 묻고 진행.

## 3. 사용자 페르소나

### 페르소나 1 — 학술 검토자 (Reviewer)

**프로필**:
- ML·통계 학술 배경 (학부 4학년 ~ 박사 후)
- 정직성 사슬·재현성·negative finding 의 학술 가치 평가
- *모델 성능* 보다 *방법론 정확성* 검증에 시간 투입

**관심**:
- 룩어헤드 차단·격리 검증 코드
- PR-AUC < base rate 정직 박제
- ablation 결과 (K=3 vs K=4·class weight·forward window)
- bootstrap CI + 통계적 변동성 명시

**불만 가능성**:
- 결과만 보여주고 *방법론은 모델 카드 별도 페이지* 인 UI → 학술 검토 흐름 단절
- 한계 페이지가 *작은 footer* 면 정직성 가치 약화

### 페르소나 2 — 도메인 전문가 (Domain Expert)

**프로필**:
- 한국 금융·재무·시장 분석 실무 5+ 년
- 종목 단위 분석 (재무비율·국면·주가) 습관
- *KOSPI200 부실 사건 희소성* 자체를 도메인 지식으로 인지

**관심**:
- 특정 종목 (예: SK 034730) 의 *국면 × 재무 × 실제 사건* 매핑
- 지주회사 fs_div (CFS vs OFS) 차이
- 2020 코로나·2008 위기 시점의 *모델 반응*
- D2 양성 20 종목의 *실제 도메인 의미*

**불만 가능성**:
- 종목 선택 UI 없으면 *데이터 dump*
- 한국어 종목명 깨짐
- 시점 매핑이 영업일 단위 아님 (분기말만)

### 페르소나 3 — 일반 평가자 (General Reviewer)

**프로필**:
- 채용 담당자·동료 개발자·일반 관심자
- ML·금융 도메인 *부분 지식*
- 5-10분 안에 *프로젝트 가치* 파악하려 함

**관심**:
- 첫 페이지에서 *프로젝트 메시지* 즉시 이해
- 사용자 시나리오 (종목 선택 → 통합 리포트) 직접 체험
- *왜 negative finding이 가치 있는가* 평이한 설명
- 모델 한계의 *책임감* 인지

**불만 가능성**:
- 한국어 마이크로카피 없으면 *전문 용어 벽*
- 첫 페이지가 *§5.5.X 박제 인용 dump* 면 진입 장벽
- "이거 어떻게 쓰는 거지?" 헷갈리는 인터랙션

### 페르소나 4 — 자기 평가자 (Self-Evaluator)

**프로필**:
- 본 프로젝트 작성자 (1인 개발자)
- 진척·결과·다음 단계 확인 목적
- 모든 박제 본문 인지 상태

**관심**:
- *어디까지 됐나* 빠른 확인
- 추가 작업 시 *현 상태 baseline* 확인
- *모든 단계 산출물* 한 화면 종합

**불만 가능성**:
- 단계별 진척이 한 페이지에 안 보이면 *컨텍스트 스위칭* 비용
- 데이터 산출물 (parquet·yaml) 직접 확인 도구 부재

## 4. 핵심 사용자 시나리오 (Use Cases)

### UC-1: 학술 검토자가 정직성 사슬 5 차원 검증

**진입**: README → 대시보드 실행 → 개요 페이지

**플로우**:
1. 개요 페이지에서 *정직성 사슬 5 차원* 즉시 시각화 확인
2. 각 차원별 *박제 위치* 클릭 → 해당 페이지 또는 코드 링크
3. Limitations 페이지에서 *6 한계 항목* 정직 박제 검증
4. 모델 카드 다운로드 또는 표시 → 재현성 정보 확인

**성공 기준**: 5분 내 정직성 5 차원 명세 + 각 박제 위치 식별

### UC-2: 도메인 전문가가 SK 034730 통합 검토

**진입**: 사이드바 → 종목 분석

**플로우**:
1. 종목 selectbox 에 "034730" 입력 또는 "SK" 검색
2. 헤더에서 *종목명 + 시총 + fs_div* 즉시 확인
3. 주가 차트에서 *시장 국면 색상 overlay* 확인 (코로나 2020-02~05 위험회피 빨강)
4. 분석 시점 selectbox → 2020-06-30 선택
5. 위험 점수 + 양성 라벨 + fs_div 확인
6. *국면 조건부 해석* 박스 읽기 — "위험회피 국면 + 영업이익 4.1조 감소" 해석
7. 재무 비율 추이 — 2020 시점 영업이익률 z-score 확인
8. 한계 경고 박스 확인 — *random 미만이라 가이드만*

**성공 기준**: 10분 내 SK 의 *국면 × 재무 × D2* 통합 해석 + 모델 한계 인지

### UC-3: 일반 평가자가 K=4 ablation 가치 이해

**진입**: 사이드바 → 시장 국면 시계열

**플로우**:
1. K=3 기본 결과 확인 (위험회피 27.9% — 코로나 시점 부합 약함 명시)
2. *K=4 ablation* 섹션 — K=3 vs K=4 *코로나 spot-check 비교* (27.9% vs 82.0%)
3. *진단 → 가설 → ablation → 정답 발견* 사이클 스토리 박스
4. log-likelihood 개선·BIC 자동 선택 차트
5. *왜 본 라인을 K=3 유지하나* 정직 박제

**성공 기준**: 5분 내 ablation 의 *과학적 가치* 이해

### UC-4: 사용자가 임의 종목 입력 → 통합 확인

**진입**: 종목 분석 페이지

**플로우**:
1. universe 321 종목 selectbox 또는 검색
2. 분석 시점 selectbox (28 분기말)
3. *위험 점수 + 국면 + 라벨* 4 metric 즉시 확인
4. *국면 조건부 해석 박스* 읽기
5. 주가 + 재무 비율 visual 확인
6. *한계 경고* 인지

**성공 기준**: 3분 내 임의 종목의 통합 정보 확인

### UC-5: 모델 한계 + Limitations 통합 이해

**진입**: 사이드바 → ⚠️ Limitations

**플로우**:
1. 페이지 진입 시 *Negative finding 정직 박제* 강조
2. 6 한계 항목 (단계 2·(A)·단계 3·HMM 시드·K tension·모집단)
3. 각 항목별 *수치·박제 위치·의미*
4. *학술 가치 메시지* 5 항목

**성공 기준**: 5-7분 내 *모든 한계의 정직성·이유·의미* 종합 이해

### UC-6: 자기 평가자가 단계 진척 확인

**진입**: 개요 페이지

**플로우**:
1. 단계 1~5 종합 표 (각 단계별 commit·산출물·박제 위치)
2. 추가 작업 후보 (Future Work 6 항목)
3. 단계별 모델 카드 링크
4. PROGRESS §5.X 박제 navigator

**성공 기준**: 2분 내 *전체 프로젝트 상태* 종합

### UC-7: 모바일·태블릿에서 비상 검토 (반응형)

**진입**: 모바일 브라우저

**플로우**:
1. 사이드바가 햄버거 메뉴로 접힘
2. 각 페이지가 1-column 단순 레이아웃
3. 차트가 모바일 크기에 맞게 자동 축소
4. 한국어 텍스트 wrap 자연

**성공 기준**: 모바일에서도 핵심 정보 (위험 점수·국면·한계) 접근 가능

## 10. 상태 처리

### 10.1 로딩 상태
- 데이터 로드 중: `st.spinner("데이터 로드 중...")` (>500ms 작업만)
- 첫 진입 시 모든 데이터 cached (`@st.cache_data`)

### 10.2 빈 상태 (Empty State)
- skipped fold 시점: "이 시점은 평가에서 제외됨 (양성 0)" 안내
- universe 비멤버 종목: "이 종목은 분석 시점 universe 멤버가 아님"
- 데이터 산출물 부재: "scripts/train_d2_baseline.py 실행 필요" + 명령 코드

### 10.3 에러 상태
- 모든 에러는 *명확한 한국어* + *복구 방법* 안내
- traceback 노출 금지 (개발 모드 제외)
- 부분 실패 (예: 일부 차트 렌더 실패) 도 *다른 섹션은 정상 표시*

### 10.4 부분 데이터
- prediction 없으면 — "skipped fold" 표시 + 차트는 계속
- regime 매핑 불가 (2015 초반) — "warmup 9 개월 안내" + 차트는 계속
