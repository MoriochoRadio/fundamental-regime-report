# 데이터 소스 (Data Sources)

본 문서는 본 프로젝트가 사용하는 모든 외부 데이터의 출처·수집 방법·
한계를 한 곳에 정리한다. *재현성·추적가능성* 을 위해 외부 시스템에서
수동 다운로드하는 모든 파일은 `data/external/` 에 두고 매니페스트에
기록한다.

---

## 1. 데이터 소스 개요

| # | 소스 | 데이터 | 수집 방식 | 위치 |
|---|---|---|---|---|
| 1 | **DART Open API** | 재무제표 | `opendartreader` 코드 자동 | `data/raw/dart/` (캐시) |
| 2 | **pykrx** | 단일 종목 일별 OHLCV·종목명 | 코드 자동 | `data/raw/pykrx/` (캐시) |
| 3 | **FinanceDataReader (FDR)** | 현 시점 KOSPI 전종목 (Marcap 포함), 상장폐지 데이터, KOSPI200 지수 시계열 | 코드 자동 | `data/raw/fdr/` (캐시) |
| 4 | **KRX 정보데이터시스템** | **시점별(분기말) KOSPI200 구성 종목** | **수동 다운로드** | `data/external/kospi200_quarterly/` (git 추적) |

## 2. 한계 및 제약 (2026-05-18 가용성 검증 결과)

본 프로젝트의 데이터 가용성에는 다음 제약이 있다. 모두 단계 1의
가용성 검증(`scripts/check_universe_availability.py`,
`scripts/check_pykrx_health.py`)에서 직접 확인된 사실이다.

- **KRX는 2014-05-01 이전 데이터를 제공하지 않는다.** 분석 기간이
  애초 2010-2024였으나 2015-2024로 단축된 사유.
- **pykrx의 *시점별 전종목·인덱스 호출* 은 모두 망가져 있다** (KRX
  응답 컬럼 변경에 라이브러리가 따라가지 못함). pykrx는 *단일 종목
  OHLCV·종목명*에만 사용한다.
- **KOSPI200 시점별 구성을 직접 API로 제공하는 라이브러리는 없다.**
  pykrx·FDR 모두 `NotImplementedError` 또는 빈 결과를 반환.
  → 본 프로젝트는 KRX 정보데이터시스템에서 *수동 다운로드*한다.
- **자동 크롤링은 채택하지 않는다.** KRX 페이지 구조 변경 시 fragile
  (pykrx가 그 사례). 1회성 수동 다운로드 + 정적 보관이 1인 포트폴리오
  에 더 안전.
- 분석 대상은 **KOSPI200 한정**. KOSDAQ·소형주는 후속 과제.

## 3. KRX KOSPI200 분기별 구성 — 수동 다운로드 절차

### 3.1 다운로드 위치

- **URL**: <http://data.krx.co.kr>
- **메뉴 경로** (2026-05-18 사용자 확인 기준):
  - 통계 > **기본통계** > 지수 > 주가지수 > **[11006] 지수구성종목**
  - (애초 메모해둔 [11005]는 현재 [11006]으로 번호가 바뀐 상태. 메뉴
    이름은 동일.)
  - 또는 좌측 검색에 "지수구성종목" 입력
- **인덱스 선택**: `코스피 200` (인덱스 코드 `1028`)
- **조회일자**: 아래 §3.3의 40개 분기말 일자
- **출력**: CSV 다운로드 (확인된 인코딩: **CP949, BOM 없음**)

### 3.2 파일명 규칙

```
data/external/kospi200_quarterly/kospi200_<YYYYQQ>_<YYYYMMDD>.csv
```

예:
```
kospi200_2015Q1_20150331.csv
kospi200_2024Q4_20241231.csv
```

- `<YYYYQQ>` = 분석 대상 분기 (예: 2015Q1)
- `<YYYYMMDD>` = *KRX가 응답에 명시한 실제 기준 영업일* (입력 calendar
  분기말이 비영업일이면 KRX가 직전 영업일로 처리; 응답 헤더의 일자를
  기록)

> **참고 — 비영업일 분기말 처리**: 2026-05-18 첫 다운로드 확인 결과
> CSV 응답 본문에 *기준일이 명시되지 않는다*. 따라서:
> - 입력 calendar 일자가 **영업일**이면 그 일자를 `actual_reference_date`
>   에 그대로 기록한다.
> - 입력 calendar 일자가 **비영업일**(예: 12-31 토요일/일요일)이면
>   KRX 검색 결과 화면 상단 또는 다운로드 다이얼로그에 표시되는
>   *적용 일자*를 그대로 `actual_reference_date`에 기록한다.
>   (KRX는 보통 *직전 영업일* 로 자동 매핑한다.)
> - 영업일 여부가 의심스러우면 `notes`에 메모만 남기고 `actual_reference_date`
>   를 `null` 로 두면 된다. 로더(`src/frr/data/universe_loader.py`,
>   후속 작업)는 `actual_reference_date` 가 비어 있으면 `calendars.py`
>   의 KRX 영업일 캘린더로 *직전 영업일* 을 자동 계산해 채운다.

### 3.4 CSV 스키마 (2026-05-18 확인분)

첫 다운로드(2015Q1)로 확인된 응답 포맷. **모든 분기에 동일하다고 가정**
하며, 새 분기에서 컬럼이 달라지면 매니페스트 `notes` 에 기록한다.

- **인코딩**: `cp949` (BOM 없음)
- **헤더**: 한 줄, 한국어
- **행 수**: 헤더 1행 + 데이터 **200~202**행 (KOSPI200은 *목표 200종목*이지만
  인덱스 리밸런싱 직후 며칠 동안 신규 편입 종목으로 일시 201/202가 발생.
  사용자 2026-05-18 일괄 다운로드로 확인).

| 컬럼명 | dtype | 비고 |
|---|---|---|
| `종목코드` | str | **6자리 문자열, 앞 0 보존** (예: `005930`). `int`로 캐스팅 금지. |
| `종목명` | str | UTF-8 한국어 |
| `종가` | float | 분기말 영업일 종가 (KRW). 매우 큰 정수가 올 수 있음 |
| `대비` | float | 전일 대비 변동, 음수 가능 |
| `등락률` | float | **퍼센트 값** (예: `0.91` = 0.91%). fraction이 필요하면 ÷100 |
| `상장시가총액` | float | **과학표기법 가능** (예: `2.12258345E8`). 단위는 KRW |

> 로더 코드는 위 dtype 표를 그대로 적용한다. 종목코드의 앞 0이 사라지면
> DART·pykrx 조회가 통째로 깨지므로 *가장 중요한 dtype 결정*은 종목코드를
> `str`로 유지하는 것이다.

### 3.5 매니페스트 작성

다운로드한 각 파일은 `data/external/kospi200_quarterly/MANIFEST.yaml`
에 엔트리가 미리 준비되어 있다. 사용자는 다운로드 후 각 엔트리의
다음 필드를 채운다:

- `filename`: 실제 저장한 파일명 (위 §3.2 규칙)
- `actual_reference_date`: KRX 응답이 명시한 실제 기준 영업일
- `downloaded_at`: 다운로드 시점 (YYYY-MM-DD)
- `sha256`: 파일의 SHA-256 해시
  - PowerShell: `Get-FileHash -Algorithm SHA256 <파일>`
  - 또는 `python -c "import hashlib; print(hashlib.sha256(open('<파일>','rb').read()).hexdigest())"`
- `encoding`: `utf-8-sig` 또는 `cp949` (열어 확인)
- `notes`: 필요 시

매니페스트 자체는 *단일 source of truth*다. 로더 코드는 매니페스트를
읽고 그 안에 *완전히 채워진 항목*만 사용한다.

## 4. 자동 수집 데이터 — 변경 시 절차

DART·pykrx·FDR로 코드가 자동 수집하는 데이터는 캐시(`data/raw/`)에
저장되고 *git 추적하지 않는다*. 다만 다음 정보는 코드 또는 별도
문서에서 추적한다:

- 라이브러리 버전 (`pyproject.toml` + `uv.lock`)
- 캐시된 응답의 수집 시점 (캐시 파일 메타데이터)
- API 키 (DART) 는 `.env` 로만 주입, 결코 커밋하지 않음
