# CLAUDE.md

이 문서는 본 프로젝트의 **변하지 않는 사실·규칙·방향**을 담는다.
가변적인 진행 상황은 `PROGRESS.md`에 기록한다.

---

## 1. 프로젝트 정체성

- **제목(가안)**: 기업 펀더멘털 + 시장 국면 인지형 통합 분석 리포트 시스템
- **성격**: 1인 개발자의 **취업 포트폴리오 프로젝트**
- **도메인 이력**: 직전 프로젝트는 AI 기반 의료영상 분석(객체탐지·도메인 적응) — 웹 구현 + GitHub 형상관리 + 논문 초안까지 완료. 본 프로젝트는 동일한 역량을 **금융 도메인**으로 이전하기 위한 작업이다.

---

## 2. 목표 (Goal)

사용자가 특정 기업을 선택하면,

1. 그 기업의 **재무 건전성·리스크 분석 리포트**를 자동 생성하고,
2. 동시점의 **시장 국면(regime)** 및 해당 기업의 **주가 궤적**을
3. 하나의 **통합 대시보드**에서 함께 보여준다.

**차별화 포인트**: 펀더멘털 리스크 해석을 **시장 국면에 조건부**로 제시한다.
(예: "동일한 부채비율 상승이라도, 위험회피 국면에서는 더 강하게 해석한다.")

---

## 3. 핵심 모듈

### 3.1 펀더멘털 모듈
- 데이터 수집: **DART Open API**로 재무제표 수집
- 산출: 재무 건전성 / 부실 위험 지표
- 모델링:
  - ML 모델로 **부실/등급 스코어링**
  - **LLM**으로 구조화된 **서술형 리포트** 생성

### 3.2 시장 국면 모듈
- 데이터: 지수 수익률·변동성
- 방법: **HMM 또는 클러스터링**으로 현재 시장 국면 **분류**
- **주가 예측이 아님.** 국면 분류·맥락 제공이 핵심이다.

### 3.3 통합 레이어
- 단일 화면에 다음을 통합:
  - 선택 기업의 주가 타임라인
  - 시장 국면 타임라인 (분류 결과를 시각화)
  - 펀더멘털 리포트
- **국면-조건부 해석**으로 리스크를 제시한다.

### 3.4 ★ AI/ML 역할 분리 (Portfolio Identity)

본 프로젝트가 *직접 학습·개발하는 AI* 는 다음 두 가지이며, 이 두 가지가
포트폴리오 평가의 **기둥**이다:

- 펀더멘털 **부실/등급 스코어링 모델** (지도학습; sklearn + LightGBM)
- 시장 **국면 분류 모델** (비지도학습; hmmlearn 본 라인, GMM/K-Means 비교)

LLM은 *직접 만든 AI* 가 **아니다**. LLM은 *이미 검증된 수치·라벨을
한국어로 서술하는 보조 도구*다. 따라서:

- LLM은 어떤 수치·라벨도 **결정하지 않는다**. 입력으로만 받는다.
- LLM 출력은 *재무 사실의 새 생성이 아니라 기존 사실의 변환*이다.
- LLM 공급자는 부품이며, `LLMProvider` 인터페이스로 추상화하여
  **교체 가능**해야 한다.
- LLM 호출은 **빌드 타임 배치**에서 1회만 발생한다. **서비스 런타임
  (대시보드)는 LLM 호출 0회** — 정적 산출물(JSON)만 읽는다.

## 4. 스코프 — 반드시 지킬 경계

### 4.1 포함 (IN)
- 시장: **한국 단일 시장**
- 데이터 소스: **DART + pykrx** 만 사용 (확정)
- 분석 대상: **고정 유니버스**의 수십~수백 개 기업
  (예: KOSPI200 또는 데이터가 깨끗한 핵심 종목 집합 — 최종 규모는 향후 결정)
- 분석 방식: **과거·배치(batch) 분석**

### 4.2 제외 (OUT) — 절대 하지 않는다
- ❌ 실시간/라이브 데이터 처리
- ❌ 정밀 주가 예측
- ❌ 실시간 트레이딩, 주문 집행, 백테스팅 기반 전략 운용
- ❌ 다국가 시장 확장
- ❌ 옵션·파생·고빈도 데이터
- ❌ 1인 개발자가 6~12주 내 끝낼 수 없는 모든 범위 확장

> **범위 보존이 모든 기술적 욕심보다 우선한다.**

## 5. 방법론적 원칙 (Methodological Rigor)

본 프로젝트는 포트폴리오로서 **방법론적 엄밀성**을 통해 평가될 것을
가정한다. 다음 원칙은 모든 모델링 단계에 적용된다.

- **데이터 누수 차단 (No data leakage)**: 학습 시점 이후의 정보는
  어떤 형태로도 학습 피처에 들어가지 않는다.
- **룩어헤드 차단 (No look-ahead bias)**: 재무제표는 **공시일 + 안전한
  지연(lag)** 이후부터 사용 가능하다고 간주한다. 시장 국면 라벨도
  시점 t 이전 데이터로만 산출되어야 한다.
- **시간 인지 분할 (Time-aware splits)**: 전통적 무작위 K-Fold 금지.
  Walk-forward / expanding window 사용.
- **재현 가능성**: 모든 데이터·모델 산출은 시드 고정 + 입력 스냅샷 기준
  재현 가능해야 한다.
- **국면 분류 결과의 사후 해석**: 국면은 사후적으로 명명될 수 있으나,
  국면 **라벨 자체는 사전적(causal)** 으로 산출되어야 한다.

## 7. 협업 규칙 (작업 방식)

본 섹션은 어시스턴트(Claude)와의 협업 방식 자체에 대한 **고정 규칙**이다.

### 7.1 문서 운영
- **CLAUDE.md (이 파일)**: 변하지 않는 사실·규칙·방향만 담는다.
  새 규칙·결정이 확정되면 반영한다.
- **PROGRESS.md**: 작업 단위가 끝날 때마다
  *현재까지 완료 / 현재 상태 / 다음 할 일 / 미해결 이슈* 를 갱신한다.

### 7.2 코드 작성 전 절차
- 코드를 바로 짜지 않는다.
- 다음 항목은 **사용자 확인 후** 진행한다:
  - 권장 기술 스택
  - 디렉터리 구조
  - 단계별 진행 계획
  - 중요한 설계 결정(데이터 모델, 모델 선택, 평가 지표, 리포트 포맷 등)

### 7.3 단계 정의 (큰 흐름)
1. 데이터 셋업 (DART / pykrx 수집·정제·캐싱)
2. 펀더멘털 모듈 (스코어링 + LLM 리포트)
3. 국면 모듈 (HMM/클러스터링)
4. 통합 대시보드
5. 마무리 (문서·재현성·발표 자료)

각 단계는 명확한 **완료 기준(DoD)** 을 가져야 하며, 단계 종료 시
PROGRESS.md를 갱신하고 git 커밋을 제안한다.

### 7.4 커밋
- 한 작업 단위가 끝날 때마다 **단위가 명확한 git 커밋**을 제안한다.
- 커밋 메시지는 *무엇을 / 왜* 가 드러나야 한다.

### 7.5 의사결정 게이트
- 다음은 사용자 확인 없이 결정하지 않는다:
  - 분석 대상 유니버스 확정 (KOSPI200 vs 다른 집합)
  - 부실 라벨 정의 (관리종목 지정? 등급 다운그레이드? 신용 이벤트?)
  - 국면 개수와 모델(HMM의 상태 수, 클러스터링 알고리즘 등)
  - 대시보드 프레임워크 (Streamlit / Dash / Next.js 등)
  - LLM 공급자 및 호출 정책 (비용·재현성·캐시 정책)

## 8. 기술 스택 (확정분만 기록)

> 확정된 스택만 이 섹션에 들어온다. 후보·검토 중인 항목은 PROGRESS.md
> 의 *결정 대기* 섹션에 둔다.

### 8.1 분석 도메인
- 데이터 소스: **DART Open API** (OpenDartReader), **pykrx**
- 시장 범위: **한국 단일 시장**
- 분석 방식: **배치(과거 데이터)**

### 8.2 언어·런타임·도구
- 언어: **Python 3.11**
- 패키지 매니저: **uv** (`pyproject.toml` + `uv.lock`)
- 포맷·린트: **ruff**
- 타입체크: **mypy** (선택적·도메인 객체 한정)
- 테스트: **pytest**
- 노트북: **Jupyter** + **nbstripout**
- 시크릿: `.env` + python-dotenv (`.env.example` 커밋)
- CI: **GitHub Actions** (lint + test만)

### 8.3 데이터·산출물
- 데이터 처리: **pandas** + **numpy** (대용량 시 polars 부분 도입)
- 저장 포맷: **parquet (Snappy)** + JSON
- 산출물 추적: 파일시스템 + git (MLflow 미도입)
- 재현성: seed 고정, 입력 스냅샷 해시, 시간 인지 분할 명시

### 8.4 모델
- ML(부실 스코어링): **scikit-learn** + **LightGBM** (캘리브레이션 포함)
- 국면 모델: **hmmlearn** 본 라인 + sklearn GMM/K-Means 비교
- LLM: **Google Gemini API (Free Tier)** — `LLMProvider` 추상화로 교체 가능
- LLM 호출 정책: **빌드 타임 1회 배치 → JSON 고정 → 런타임 호출 0회**

### 8.5 프레젠테이션
- 대시보드: **Streamlit**
- 시각화 (대화형): **plotly**
- 시각화 (정적·리포트): **matplotlib**

## 9. 안티-패턴 (Do Not)

- "이것도 해두면 좋을 것 같아서" — 범위 확장 금지.
- "어차피 비슷하니까" — 시간 인지 분할 없이 평가 금지.
- "그냥 예측해보자" — 주가 예측 모듈 추가 금지.
- "당장은 잘 안 되지만 LLM에 넣으면 되겠지" — 수치 산출 책임은 ML/통계
  단에 있다. LLM은 **이미 검증된 수치의 서술화**를 맡는다.
- "실시간으로 보여주면 멋있을 텐데" — 본 프로젝트는 배치다.
- "대시보드에서 LLM을 바로 부르면 더 자연스럽지" — 금지. 서비스 런타임
  LLM 호출 0회 원칙(§3.4)을 깨면 비용·재현성·평가 모두 무너진다.
